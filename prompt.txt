You are working on Yourpreneur Canvas — a founder-focused platform where entrepreneurs record, track, and visualize their ventures on an interactive timeline canvas. Ventures appear as nodes on a horizontal time axis, positioned by date, with events logged beneath each node. The platform syncs data from external tools (Notion, GitHub, Linear, Vercel, Product Hunt) and displays everything as a living chronicle.
Your job is to audit the entire codebase, understand the current structure, and apply the following fixes. Do not break existing functionality. Where functionality is missing or incomplete, implement it yourself.

1. DESIGN SYSTEM — UNIFY EVERYTHING

Audit all components and enforce a single design token system: one background color scale, one text scale, one border radius, one spacing scale, one color palette for statuses (ACTIVE = teal, PIVOT = amber/coral, ARCHIVED = grey, ACQUIRED = gold)
All cards, panels, sidebars, modals must share the same border style, corner radius, and background opacity
Typography must be consistent — one font, defined size scale (xs/sm/base/lg/xl), no random sizing
All icon sizes must be uniform across toolbar, sidebar, and feed
Fix all grammar bugs — "1 EVENTS" → "1 EVENT", "2 EVENTS" stays — apply pluralization logic globally
Remove or replace any placeholder/test data visible in the UI (e.g. entries named "Fuckedup", "N" initials, "UNASSIGNED" labels)
The bottom toolbar icons must all be the same size, weight, and style — audit and normalize them


2. CANVAS

The timeline rail must span the full canvas width with consistent month labels (JAN, FEB... DEC) at equal intervals — never cut off at edges, always fade out gracefully with a gradient mask
Every node card on the canvas must have a visible, clean vertical line connecting it to its exact dot position on the timeline rail — no floating cards
Nodes positioned above the timeline = pre-launch / ideation phase. Nodes below = active / launched. Add a faint label on left edge of canvas: "IDEATION" above the rail, "ACTIVE" below. This makes the vertical axis meaningful
All node cards must be the same base size — do not scale them by zoom proximity. Zoom affects the canvas, not individual card sizes
Orphaned timeline dots (dots with no connected card) must either show a tooltip on hover explaining what they are, or be removed if they serve no purpose
The large year watermark (e.g. "2024", "2025") in the canvas background is good — keep it, ensure it doesn't interfere with card readability
Canvas must support free drag of node cards. Dragging a node horizontally updates its date. Dragging vertically moves it above/below the rail
Add subtle edge fade (gradient mask) on all four sides of the canvas to communicate it is scrollable/pannable in all directions
Add a "Fit to Screen" button next to the zoom percentage indicator — clicking it resets zoom and centers all nodes


3. RIGHT PANEL — UNIFIED CONTEXT PANEL
This is the most important structural change. Replace the current separate Live Feed with a unified right-side context panel that has two states:
Default state (no node selected):

Show Live Feed (synced + auto events in a scrollable list)
Each feed item must show: venture avatar, venture name, event title, date, source (via notion / via github etc.)
Every feed item must have a consistent layout — no missing dates, no missing sources
"MOMENTUM" section: implement it — show a simple spark chart of events logged per week over last 8 weeks. If no data, show empty state message "Log events to see momentum" not a blank section
SYNCED and AUTO counters at top of panel are good — keep them

Selected state (user clicks a node on canvas):

Panel slides to show that venture's detail view
Top: venture name, category badge, status badge, health score with sparkline, sync source
Middle: scrollable event list for that venture — each event shows icon ($ for funding, 🚀 for launch, 📦 for product, ⚙️ for ops), title, date, source
"+ ADD EVENT" button opens an inline form directly in this panel — fields: event type (dropdown), title, date, notes, source. Submitting adds it to the timeline and live feed
Bottom: venture-specific TASKS section — a simple checklist. Each task has: title, due date, status (TODO / IN PROGRESS / DONE). "Add task" inline. Tasks are stored per venture
A close/back arrow at top of panel returns to default Live Feed state
This means users never need a separate modal or page to log events or manage tasks — everything happens in this panel


4. SIDEBAR

Fix the bottom user indicator — it shows "N" with "Last activity: Xm ago" — replace "N" with the actual user avatar initial pulled from the founder profile, styled consistently with other avatars in the app
"Find a venture" search in sidebar: scope it to filter the sidebar portfolio list AND highlight matching nodes on the canvas. Differentiate it from global search in the top bar (global search searches everything including events)
Portfolio list items: clicking one must pan and center the canvas to that venture's node with a smooth animation
Event count next to each venture in sidebar must reflect a timeframe — either total or "this year" — add a small label so it's clear. Example: "2 EVENTS THIS YEAR"
Sidebar collapse arrow (top right of sidebar) must work — collapse sidebar to icon-only rail, expand on click


5. NODE CARD DETAIL

Every node card must have a consistent layout: category label top left, avatar top right, venture name large, health score + sparkline, status badge, sync source label
Health score must be a real calculated value — define the formula: (events logged in last 30 days × 10) + (sync active ? 20 : 0) + (tasks completed ratio × 30), capped at 100. Implement this
Clicking a node on canvas: card does a subtle scale-up (1.0 → 1.04), right panel switches to that venture's context (see section 3)
Do not open a floating modal over the canvas — all detail lives in the right panel
"Spinoff" relationship label between nodes is good — also support "Pivot", "Acquired by", "Merged with" — render as a labeled curved line between the two related nodes


6. HEADER

"PRIVATE" button must toggle the canvas between private and public shareable view — implement share URL generation when switched to public
Zoom percentage must always reflect actual canvas zoom — clicking it resets to 100%
The sync/history icon next to zoom must open a small dropdown showing last 5 sync events with timestamps
"PROFILE PANEL READY" / "BILLING WORKSPACE READY" / "SYNCED 2M AGO" status messages in header subline — these are contextual, keep them but ensure they cycle or update dynamically based on actual app state, not hardcoded strings
Notification bell must show a badge count when there are unread live feed items


7. TASK CANVAS MODE

"TASK CANVAS" button in header switches the main canvas to a task-focused view
In this mode, each venture node expands to show its task list inline on the canvas
Tasks can be dragged between ventures
A global task list appears in the right panel showing all tasks across all ventures, filterable by status and venture
Switching back to Chronicle mode returns the canvas to the timeline view — preserve zoom and pan position


8. TOOLBAR

All toolbar buttons must have keyboard shortcuts — implement them and display them in a tooltip on hover (e.g. "NEW · N", "FOCUS · F", "AXIS · A")
"HOVER FOR KEYS" hint next to CANVAS TOOLS — clicking it should open a keyboard shortcut legend overlay
LOG tool: opens the right panel ADD EVENT form for the currently selected venture, or prompts to select a venture first if none selected
COACH tool: opens an AI-powered suggestion panel in the right panel — "Based on your last activity, here's what to focus on this week" — implement with a simple heuristic (e.g. if no events logged in 14 days, suggest logging a status update)
REVIEW tool: generates a text summary of the selected year's activity across all ventures — render it in the right panel as a readable chronicle entry


9. ONBOARDING — EMPTY STATES

If a user has 0 ventures: canvas shows a centered prompt "Start your chronicle — add your first venture" with a large "+ NEW VENTURE" button in the middle of the canvas
If a venture has 0 events: its node shows a pulsing ring and a tooltip "Log your first event for [venture name]"
If no integrations connected: Live Feed shows "Connect Notion, GitHub, or Linear to start syncing events automatically" with integration icons as buttons


10. GENERAL CODE QUALITY

Remove all hardcoded placeholder strings from the UI
All dates must format consistently: "Mar 1, 2025" — no mixing of formats
All async data fetches must have loading states — show a skeleton loader, not a blank space
All interactive elements must have hover states — buttons, cards, feed items, toolbar icons
Test the full flow: create venture → log event → view on canvas → view in feed → add task → complete task → check health score updates
