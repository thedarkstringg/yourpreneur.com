12. INTEGRATION SYSTEM
Integrations settings page /settings/integrations:

Card for each supported integration: Notion, GitHub, Linear, Vercel, Product Hunt
Each card: integration icon + name + description of what it syncs + CONNECT / DISCONNECT button + "Last synced: X" timestamp if connected
FREE: only 1 integration allowed. Other cards show lock icon + upgrade prompt after first connection

Per integration, implement the sync logic:
Notion: OAuth connection. Syncs: page titles as events, database entries as milestones. Maps page creation date to event date.
GitHub: OAuth or PAT. Syncs: repository creation, releases (tags), closed milestones, README updates. Each becomes an event.
Linear: OAuth. Syncs: completed cycles (sprints), shipped milestones, project status changes.
Vercel: OAuth or token. Syncs: successful deployments, custom domain additions, team member additions.
Product Hunt: API key. Syncs: product launch date, upvotes milestones (e.g. 100, 500, 1000 upvotes), featured status.
Sync behavior:

On connect: pull last 90 days of history and create events for them, marked as source = integration name
Ongoing: poll every 30 minutes OR use webhooks where available
Each synced event is marked as AUTO in the live feed
Manually created events are marked as MANUAL — this distinction is always visible
No sync ever overwrites manual data
